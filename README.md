# Tech Stocks Market Analysis · Malloy

> Interactive market intelligence dashboard built with [Malloy](https://www.malloydata.dev/) and DuckDB — analyzes 17 tech tickers across 5 years of daily OHLCV data.

---

## What this does

| Analysis | Description |
|---|---|
| **Top 3 / Worst 3 — Week** | Fastest growing and worst performing tickers over the last 7 days |
| **Top 3 / Worst 3 — Month** | Same ranking over the last 30 days |
| **Top 3 / Worst 3 — Year** | Same ranking over the last 12 months |
| **Projected Upside** | 30-day momentum signal: avg daily return × volume confirmation |
| **Projected Downside** | Risk signal: negative momentum amplified by volatility |
| **Full Scorecard** | All 17 tickers ranked by 5-year total return with volatility |
| **Volume Surge Detection** | Days trading >2× avg volume — unusual activity flag |

## Setup

### 1. Prerequisites

- [VS Code](https://code.visualstudio.com/)
- [Malloy VS Code extension](https://marketplace.visualstudio.com/items?itemName=malloydata.malloy-vscode) — install from the Extensions panel
- Python 3.9+

### 2. Clone the repo

```bash
git clone https://github.com/YOU/tech-stocks-malloy
cd tech-stocks-malloy
```

### 3. Download the dataset

1. Go to the Kaggle dataset linked above (free account required)
2. Click **Download**
3. Unzip and place the CSV at:

```
data/all_stocks_5yr.csv
```

### 4. Prep the data

```bash
pip install -r requirements.txt
python prep_data.py
```

This filters the raw CSV to tech tickers only and writes `data/tech_stocks.parquet` (~2 MB, fast DuckDB queries).

### 5. Open the notebook

In VS Code, open `tech_stocks.malloynb`. The Malloy extension renders each query block as an interactive table/chart inline.

You can also run individual queries from `tech_stocks.malloy` by placing your cursor in a `run:` block and pressing **Ctrl+Enter** (or **Cmd+Enter** on Mac).

## Key Malloy concepts used

### Source with reusable measures

```malloy
source: stocks is duckdb.table('./data/tech_stocks.parquet') extend {
  measure: avg_close       is close.avg()
  measure: avg_daily_return is daily_return_pct.avg()
  measure: avg_volatility   is daily_range_pct.avg()
}
```

Measures are defined once and reused across every query — no copy-pasting SQL aggregations.

### Filtered aggregates (inline WHERE)

```malloy
run: stocks -> {
  group_by: ticker
  aggregate:
    recent_avg is avg(close) { where: date >= @2018-01-31 }
    prior_avg  is avg(close) { where: date >= @2018-01-24 and date < @2018-01-31 }
}
```

Malloy lets you apply different filters to different aggregates in a single query — impossible in standard SQL without multiple subqueries.

### Pipeline queries (→ chaining)

```malloy
run: stocks -> {
  group_by: ticker
  aggregate: recent_avg, prior_avg
} -> {
  select:
    ticker
    week_chg_pct is round((recent_avg - prior_avg) / prior_avg * 100, 2)
  order_by: week_chg_pct desc
  limit: 3
}
```

The `->` operator pipes the output of one stage into the next — like method chaining, but for analytical queries.

### Named views

```malloy
view: by_ticker_year is {
  group_by: ticker, year
  aggregate: avg_close, total_volume, avg_daily_return
}
```

Named views on a source act like saved queries you can reference anywhere: `run: stocks extend { view: by_ticker_year } -> by_ticker_year`

## Adapting to a live dataset

To use a live dataset instead of the static Kaggle file, swap the source definition:

```malloy
// DuckDB with a fresh CSV
source: stocks is duckdb.table('./data/my_fresh_data.parquet') extend { ... }

// Or point at a MotherDuck cloud table
source: stocks is duckdb.table('md:my_db.tech_stocks') extend { ... }
```

All queries in the notebook stay identical — only the source line changes.
